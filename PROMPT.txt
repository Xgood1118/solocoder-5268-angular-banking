需求：Angular + Go 全栈个人银行系统，账户 / 转账 / 对账 / 脱敏 / 审计 / 限额 / 双因子认证。

背景：组里有同学想完整实现一个银行核心系统的小型版本（不是真银行，是 demo / 教学）。重点不是真接银行接口，是把"账户余额计算、转账幂等、对账一致性、敏感数据保护、审计日志、限额风控、双因子"这些银行业务核心概念用代码完整跑通。学习目标：让组员理解金融系统设计的常见坑。

核心功能：
- 账户：开户、销户、冻结 / 解冻、余额、币种、账户类型（活期 / 定期）
- 转账：同行 / 跨行（mock 跨行走"清算接口"）、实时到账 / 普通到账、附言
- 对账：每日凌晨跑批对账（系统账 vs 流水账），不平生成对账差异报告
- 脱敏：身份证 / 手机号 / 卡号显示要脱敏（仅后 4 位），日志中也要脱敏
- 审计日志：所有操作（登录 / 转账 / 改密 / 绑卡）写审计流水，存 7 年
- 限额：单笔限额、单日限额、月度限额；超限拒绝
- 双因子：登录、修改密码、绑卡走短信 / 邮箱验证码
- 利率：活期 / 定期按日计息，月末结息
- 报表：日 / 周 / 月账户余额表、流水表

涉及模块：
- `backend/internal/account/`：账户服务、余额计算
- `backend/internal/transfer/`：转账服务、幂等控制、流水
- `backend/internal/audit/`：审计日志、AOP 拦截
- `backend/internal/limit/`：限额规则、风控
- `backend/internal/recon/`：对账引擎、差异报告
- `frontend/src/app/`：登录、账户、转账、流水、对账、设置
- `frontend/src/app/core/`：双因子弹窗、确认对话框

注意事项：
- 余额计算用"借贷记账法"：每笔交易写借贷流水，余额 = SUM(credit) - SUM(debit)，不直接 UPDATE 余额字段
- 转账幂等：客户端带 `biz_id`（UUID），服务端用 `INSERT ... ON CONFLICT (biz_id) DO NOTHING`，重复请求直接返回之前结果
- 余额不能为负：`UPDATE accounts SET balance = balance + ? WHERE id = ? AND balance + ? >= 0`，affected_rows = 0 说明超限
- 并发转账：用乐观锁 + 事务隔离级别 `SERIALIZABLE`，避免 ABA 问题
- 跨行转账走"清算接口"（mock）：先冻结 → 发清算 → 收到回执 → 确认；任一环节失败要回滚
- 对账差异处理：自动对账（系统 + 流水完全一致）vs 人工对账（不一致生成工单）
- 利率计算：日利率 = 年利率 / 360（行业惯例），按日计息月末结
- 限额用 Redis 计数器（INCR + EXPIRE），实时校验比 DB 聚合快
- 双因子验证码：6 位数字、5 分钟有效、3 次错误锁定、不能连续两次相同
- 敏感字段日志脱敏：用 Jackson 注解 + Logback PatternLayout，不要自己 replace
- 审计日志要"不可篡改"：单向往对象存储写、HMAC 校验、定期归档
- 密码：bcrypt cost=12 + pepper（服务端密钥拼接）
- 跨日切账用 cron 0 点跑，要加分布式锁（Redis SETNX）防重复
- 长时间操作（跨行清算）用消息队列异步 + 状态机推进，不要 HTTP 长连接
- 前端金额格式化用 `DecimalPipe`，不要用 toFixed
- 银行风格 UI 配色：深蓝 / 金色，字体偏大，重要操作有确认弹窗
